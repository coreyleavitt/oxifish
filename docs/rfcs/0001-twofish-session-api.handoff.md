## Breaking changes 0.1.x → 0.2.0 (derived 2026-07-09 by diffing `git show HEAD` vs working tree; for the release body)
- **Removed:** all 5 mode classes (`TwofishECB/CBC/CTR/CFB/OFB`), all 6 streaming classes (`TwofishCBCEncryptor/Decryptor`, `TwofishCTRCipher`, `TwofishCFBEncryptor/Decryptor`, `TwofishOFBCipher`), `pad()`/`unpad()`/`PaddingStyle` (padding now integrated via `padding=`), `BlockSize`/`KeySize` enums. Replacements: `TwofishKey` (+ factories) / `TwofishSession` / `Mode` / `Padding` / `EncryptResult` / `DecryptionError` / `Buffer`.
- **Semantic:** (1) **LEAD ITEM — old streaming-CFB ciphertext from sub-16B `update()` chunks is undecryptable by any correct CFB** (0.1.x keystream-reuse bug; decrypt with 0.1.x before upgrading; old one-shot/aligned CFB unaffected); (2) ECB gated behind `ecb_*` factories, `"ecb"` string rejected, padding mandatory there; (3) iv keyword-only + omitting iv on encrypt returns `EncryptResult` not bytes; (4) sessions single-use + `finalize()` required (padded decrypt withholds final block; finalized reuse → RuntimeError); (5) padded-decrypt failure → `DecryptionError` (ValueError subclass — old handlers still catch; old message-string matching breaks); (6) zeros-unpad scoped to final block (old `unpad` stripped whole buffer).
- **Not breaking:** wire format unchanged except CFB item above (old CBC/CTR/OFB/ECB ct decrypts with matching `padding=`); Python floor still >=3.11; `BLOCK_SIZE` kept; throughput ±2%.
